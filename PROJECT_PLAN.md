# IKR 项目计划书

> IKR — 智慧知识库
> 让"读到"和"用到"之间的时间差，从几个月压缩到几小时

---

## 一、我们在解决什么问题

### 时代背景

AI 时代信息生产速度远超人类消化速度。每个人每天都面对海量的文章、视频、播客——质量越来越高，数量越来越多，但人的时间和注意力是固定的。

结果是：**几乎所有人都活在 FOMO 里**。

### 但真正的问题不是信息太少

现有工具（Notion、Obsidian、Readwise）都在解决"存"的问题。用户存了很多，但遇到真实问题时，依然不知道该调取哪些知识、怎么组合运用。

**知识管理工具的核心死穴是时间差：**

```
今天读到一篇好文章
        ↓
收藏、点赞、转发
        ↓
两周后遇到相关问题
        ↓
完全想不起来曾经读过
        ↓
重新搜索，或者靠直觉决策
```

这个时间差，让几乎所有的阅读和学习投入都打了水漂。

### 更深的矛盾

所有知识管理产品都假设用户会"平时积累、偶尔用"——但这中间的时间差，正是留存的死穴。用户凭什么在感受不到价值的时候，持续做积累这件事？

## 二、IKR 的 PMF

> **让每天既要大量学习、又要持续决策的专业人士，把"读到"和"用到"之间的时间差从几个月压缩到几小时。**

### 目标用户

不是"想学习的人"，而是**每天既在大量吸收信息、又在持续解决问题的专业人士**：创业者、产品经理、咨询顾问、投资人、管理者……

这类用户的关键特征：
- 内容消费和问题决策**发生在同一个时间密度里**，时间差天然很短
- 知道信息质量直接影响决策质量，愿意为"决策更好"付费
- 不需要培养新习惯——他们本来就在读、本来就在做决策

### 核心场景

```
早上：刷到一篇关于用户增长的文章，转发给 IKR（10秒）
        ↓
IKR 后台自动消化，提炼核心观点，纳入知识库
同时：自动找到相关源头资料作为参考层
        ↓
下午：开会讨论增长策略，顺手问 IKR（1分钟）
        ↓
IKR 基于今早那篇文章 + 历史积累，给出多种方案和推荐
        ↓
用户在会议上提出了一个有依据的判断
```

**这个闭环在同一天内完成。** 用户立刻感受到积累的价值，形成正向强化——每次提问验证积累的意义，每次积累提升下次提问的质量。

## 三、产品是什么

**IKR 是你的个人决策顾问。**

它了解你读过什么、消化过什么。当你遇到问题时，基于你自己的知识体系，给出有依据的多种解决方案。

> 核心主张：**转发就够了。遇到问题时，IKR 告诉你怎么做。**

用户感知到的不是"我在管理知识"，而是**"我做决策更快、更有底气了"**。

认知提升是这个过程的自然结果——不是产品的承诺，而是用户用了一段时间后自己发现的变化。

### 与 NotebookLM 的本质区别

NotebookLM 是文档问答工具：用户主动上传文件，AI 帮你分析，用完即走，没有积累，没有个性化。

IKR 是会成长的个人顾问：知识来自日常碎片阅读，持续积累，越用越懂你，用得越久越难被替代。

| | NotebookLM | IKR |
|--|-----------|-----|
| 内容来源 | 用户手动上传文件 | 日常碎片内容随手转发 |
| 时间维度 | 静态，上传完即固定 | 动态，持续生长 |
| 核心价值 | 帮你读懂一批文档 | 把日常阅读转化为决策能力 |
| 护城河 | 无 | 个人知识积累越深越难迁移 |

## 四、产品核心机制

### 机制一：自动消化（零负担建立知识基础）

用户转发一篇文章，IKR 在后台自动完成：

- **结构化摘要**：核心论点、关键概念、作者逻辑链，直接呈现
- **原子化知识卡片**：拆解为独立知识点，成为后续调用的最小单元
- **语义关联**：自动与库里已有内容建立联系，形成知识网络
- **源头补充**：自动寻找相关的更专业、更源头的资料（原始研究、经典文献），作为独立的参考层，不混入个人知识库

用户唯一的动作是转发。消化、整理、关联，全部由系统完成。

### 机制二：问题解答（核心价值，即时兑现）

**核心要求：回答必须可实操——用户看完知道第一步做什么。**

三层机制保障实操质量，按优先级依次触发：

**第一层：从知识库推断用户背景（自动，无感）**
用户积累的内容透露了行业、角色、所处阶段。每次提问前自动构建用户画像，同一问题对不同用户给出不同的实操路径。

**第二层：每个方案内置实操四要素（始终执行）**
- 适用前提：满足什么条件选这个方案
- 具体步骤：第一步→第二步（2-4步，可执行）
- 判断标准：怎么知道做对了
- 知识来源：来自知识库 / 通用知识

**第三层：问题模糊时问一个澄清问题（谨慎触发）**
仅当无法区分实操路径时触发，给出选项，降低回答成本。

回答结构分两层：
- **你的知识库**：来自用户自己转发的内容，是主要依据
- **延伸参考**：系统自动找到的源头资料，独立呈现

**冷启动**：知识库不足时以通用知识兜底，实操结构不变，适用前提更宽泛，同时提示积累进度。

### 机制三：知识留存（静默运行，不制造负担）

间隔重复有充分的认知科学依据，但要求用户主动参与的产品大众坚持率极低。IKR 让它完全静默：

- 后台根据遗忘曲线自动调度，用户无感知
- 当用户提问时，系统优先调用需要强化的知识卡片构建回答——复习在"解决问题"的过程中自然完成
- 没有复习任务，没有打卡，没有债务感

> 复习不是用户的任务，是系统提升答案质量的手段。

### 三个机制的飞轮

```
用户转发内容（10秒）
        ↓
[后台] 自动消化 → 进入个人知识库
[后台] 源头补充 → 作为参考层
        ↓
用户遇到问题，向 IKR 提问
        ↓
[核心] 基于个人知识库 → 多方案 + 推理过程 + 分层回答
        ↓
用户解决了问题，感受到积累的价值
        ↓
更愿意继续转发内容 ──────────────────┐
        ↓                            │
知识库越来越个性化                    │
        ↓                            │
回答越来越准确 ───────────────────────┘
```

用得越久，越难被替代。这是 IKR 的护城河。

## 五、用户体验设计原则

### 原则一：用户的唯一主动动作是转发和提问

消化、分类、关联、调度——全部由系统完成。不需要整理，不需要打标签，不需要建目录，不需要维护知识图谱。

### 原则二：微信是入口，不是主战场

微信适合做内容收集入口——随手转发，零摩擦。深度交互在 Web/App 完成。

- **微信负责**：文章转发收录
- **Web/App 负责**：提问、查看回答、浏览知识库

### 原则三：让用户感受到"决策更好了"，而非"学了很多"

没有"今日待复习"，没有打卡，没有知识点完成率。

有的是：开会前问了一个问题，得到了三种有依据的方案，其中一种来自昨天看的文章——然后会议上真的用上了。

## 六、目标用户

### 核心用户画像

**每天既要大量学习、又要持续决策的专业人士**

- 创业者、产品经理、咨询顾问、投资人、管理者
- 每天接触大量行业信息，同时每天面对大量需要判断和决策的问题
- 知道信息质量影响决策质量，有为此付费的意识
- 痛点：读了很多，但做决策时还是靠直觉，感觉没有真正用上

### 用户分层

| 层级 | 特征 | IKR 对他们的价值 |
|------|------|-----------------|
| 核心用户 | 高频内容消费 + 高频决策，时间差天然短 | 飞轮转得最快，价值感知最强，最容易留存 |
| 主流用户 | 高频内容消费，决策频率中等 | 在关键决策时刻体验到价值，逐渐建立习惯 |
| 潜在用户 | 偶尔读文章，决策靠经验 | 需要教育成本，暂不作为主要目标 |

## 七、商业模式

### 免费 + 订阅

| 免费版 | 付费版 |
|--------|--------|
| 每月 20 篇内容 | 无限内容 |
| 基础摘要 | 深度消化 + 知识图谱 |
| 每月 20 次提问 | 无限提问 + 多方案对比 |
| 通用 AI 兜底 | 个性化知识库权重 + 源头补充 |

### 定价逻辑

用户付费买的是"决策更好"，而不是"学了很多"。对专业人士来说，一次更好的决策带来的价值，远超订阅费用。

参考区间：¥49-99/月，¥399-699/年。（对标专业工具，而非学习类 App）

## 八、里程碑规划

### Phase 0：验证 PMF
**目标**：证明"同一天内读到用到"的闭环能真实发生

- [ ] AI 消化链路：转发文章 → 自动生成摘要 + 知识卡片
- [ ] 微信服务号接入（内容入口）
- [ ] 核心问答：基于知识库检索 + 多方案 + 来源标注 + 分层回答
- [ ] 冷启动：通用 AI 兜底 + 积累进度可视化
- **验证指标**：种子用户同一天内"转发 + 提问"都发生的比例 > 40%；反馈"比直接问 ChatGPT 更有用"

### Phase 1：MVP 上线
**目标**：完整体验可用，开始规模化验证

- [ ] 用户账户体系
- [ ] Web 端提问和知识库界面
- [ ] 知识留存静默调度
- [ ] 源头资料自动补充
- [ ] 视频内容支持
- **目标**：100 个付费用户，月留存 > 65%

### Phase 2：飞轮加速
**目标**：让"用得越久越好用"的飞轮效应可感知

- [ ] 知识图谱可视化
- [ ] 回答质量随积累提升的可视化
- [ ] 多方案深度对比
- **目标**：月留存率 > 70%，NPS > 50

### Phase 3：规模化

- [ ] 浏览器插件

## 九、核心风险与应对

| 风险 | 描述 | 应对 |
|------|------|------|
| 冷启动流失 | 知识库为空时体验差，用户在感受到价值前流失 | 通用 AI 兜底保证第一天有价值；积累进度可视化让用户看到增长 |
| 使用频率不稳定 | "遇到问题才来"导致日活低 | 核心用户每天都有决策需求，频率天然高；不追求日活，追求"关键时刻在场" |
| 仪式感陷阱 | 用户感觉有用但没有真实价值，最终流失 | 核心指标是"用 IKR 的回答解决了实际问题"，而非打开率 |
| 竞争对手跟进 | ChatGPT、Notion AI 等做类似功能 | 个人知识库的个性化积累是护城河——通用 AI 永远不了解你读过什么 |
| 回答质量参差 | AI 生成的方案不够实用 | 精细 Prompt 工程；用户隐式反馈持续优化 |

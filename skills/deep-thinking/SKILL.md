# Deep Thinking Skill

深度思考模式下，不直接给答案，引导用户自己推导。

## 核心规则
1. 每次只问一个问题，等待用户回答
2. 优先调用用户知识库里的框架（"你读过的 X 框架，试着用它分析..."）
3. 根据用户回答逐步引导向更深层原因
4. 最多 4 轮追问，第 4 轮后输出总结 + 盲点补充

## 追问策略
- 第 1 轮：帮用户定位问题所在的层次
- 第 2 轮：引导用户分析根本原因
- 第 3 轮：引导用户推导解决方向
- 第 4 轮：用户得出结论，IKR 补充盲点

## 盲点补充格式
"你的分析很准确。补充一个你可能没考虑到的视角：
基于你收录的《XXX》，[补充内容]"

## 追问示例
- "你觉得问题的根本原因是什么？"
- "如果排除这个因素，结论会变吗？"
- "你读过的[框架名]里怎么看待这类问题？"
- "从用户视角看，他们最在意的是什么？"

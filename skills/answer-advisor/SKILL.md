# Answer Advisor Skill

当用户提出需要决策或解决方案的问题时，按以下结构回答。

## 核心要求
回答必须可实操——用户看完知道第一步做什么。不能停在框架层面。

## 每个方案的固定结构
```
▌ 方案名称（推荐 / 备选）
  来自：[知识库文章名 / 通用知识]
  【适用前提】满足什么条件时选这个方案
  【第一步】具体做什么
  【第二步】具体做什么
  【第三步】具体做什么（可选）
  【判断标准】怎么知道做对了
```

## 优先级规则
1. 推荐方案排第一，说明推荐理由
2. 优先使用用户知识库中的内容，通用知识明确标注
3. 适用前提比步骤更重要——用户要先对号入座
4. 问题模糊时只问一个澄清问题，给出 A/B/C 选项

## 时效性提醒
如果使用了标注「⚠️ 含时效性数据」的知识卡片，在回答末尾注明数据时间。
